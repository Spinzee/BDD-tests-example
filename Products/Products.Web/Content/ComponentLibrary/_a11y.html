Accessibility (A11y)

Make sure you test all pages at minimum using:

  • Keyboard only navigation
  • Screen reader (such as ChromeVox and/or iPhone speech)

General guidelines

If you design for those who have the hardest time on our sites - keyboard only users, voice only users etc - you should end up with a product which is easier for all your users.

  • Make sure any button taken out of context has hidden screen-reader text. e.g. [buy] button should have "buy x product at y price" when listened to by a screen reader. Example: buttons on OJB slot booking calendar page
